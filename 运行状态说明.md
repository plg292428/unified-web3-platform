## 🧪 测试步骤

### 1. 检查服务状态

运行以下脚本检查服务是否正在运行：
```powershell
.\check_running_status.bat
```

### 2. 访问Swagger UI

在浏览器中打开：
```
https://localhost:7266/swagger
```

**应该看到：**
- API端点列表
- 每个端点可以展开查看详情
- "Try it out"按钮可以测试端点

### 3. 测试健康检查

在浏览器中访问：
```
http://localhost:5195/health
```

**预期响应：**
```json
{
  "status": "healthy",
  "timestamp": "2025-01-27T..."
}
```

### 4. 测试数据库连接

在Swagger UI中测试需要数据库的API端点，例如：
- 用户相关端点
- 数据查询端点

验证数据库连接和操作是否正常。

## ⚠️ 注意事项

### 启动时间

项目首次启动可能需要：
- **10-30秒**：加载依赖和初始化服务
- 如果超过1分钟仍未启动，可能有错误

### 查看运行日志

如果使用 `.\run_backend.bat` 启动，控制台会显示：
- 启动信息
- 错误信息（如果有）
- 监听地址

### 停止服务

**方法1：** 如果在前台运行，按 `Ctrl+C`

**方法2：** 如果后台运行，关闭窗口或：
```powershell
taskkill /F /IM dotnet.exe
```

## 🔍 故障排除

### 如果服务无法启动

1. **检查编译错误**
   ```powershell
   cd src\Backend
   dotnet build
   ```

2. **检查数据库连接**
   - 确保SmallTarget数据库存在
   - 验证连接字符串

3. **检查端口占用**
   - 确保7266和5195端口未被占用

### 如果无法访问Swagger UI

1. **检查服务是否启动**
   - 运行 `.\check_running_status.bat`

2. **检查HTTPS证书**
   - 开发环境可能显示证书警告
   - 点击"高级" → "继续访问"

3. **尝试HTTP端点**
   - `http://localhost:5195/swagger`
